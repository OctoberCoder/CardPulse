Data Flow

How a Card Sell Transaction Flows
1. User submits card → POST /api/cards/submit (brand, code, denomination)
2. Rate engine quotes price → dynamic rate from brand/denom table
3. User accepts quote → card locked, admin verification step
4. Admin approves → card balance confirmed, payment queued
5. User receives payout → wallet balance updated / bank transfer triggered
